# Developer Setup Guide

## Prerequisites

| Tool | Version |
|---|---|
| PHP | 8.3 or higher |
| Composer | 2.x |
| Node.js | 20+ (LTS recommended) |
| npm | 10+ |
| Git | Any recent version |

For production or MySQL development, you also need:
- MySQL 8.0+ or MariaDB 10.6+

---

## Quick Start (SQLite — recommended for development)

### 1. Clone the repository

```bash
git clone https://github.com/armannadim2/copyus_shop.git
cd copyus_shop
```

### 2. Install PHP dependencies

```bash
composer install
```

### 3. Configure environment

```bash
cp .env.example .env
php artisan key:generate
```

### 4. Run database migrations

```bash
php artisan migrate
```

Optionally seed demo data:

```bash
php artisan db:seed
```

### 5. Install and build frontend assets

```bash
npm install
npm run build
```

For live reloading during development, use:

```bash
npm run dev
```

Keep this terminal open alongside your PHP server.

### 6. Start the development server

```bash
php artisan serve
```

The application is now available at [http://localhost:8000](http://localhost:8000).

## Environment Variables

Copy `.env.example` to `.env` and fill in the values below.

### Required

```env
APP_NAME="CopyUS Shop"
APP_ENV=local
APP_KEY=                     # generated by `php artisan key:generate`
APP_URL=http://localhost:8000
APP_LOCALE=ca                # ca | es | en
```

### Database (SQLite — default for development)

```env
DB_CONNECTION=sqlite
# DB_DATABASE is auto-resolved to database/database.sqlite
```

### Database (MySQL — production / staging)

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=copyus_shop
DB_USERNAME=root
DB_PASSWORD=secret
```

### Shop Behaviour

```env
SHOW_PRICES=true   # Set to false to hide prices from non-B2B guests
```

### AI Content Generation

Choose one provider and fill in its credentials:

```env
AI_PROVIDER=gemini            # or 'anthropic'

# Google Gemini
GEMINI_API_KEY=your-key-here
GEMINI_MODEL=gemini-2.0-flash

# Anthropic Claude
ANTHROPIC_API_KEY=your-key-here
ANTHROPIC_MODEL=claude-haiku-4-5-20251001
```

### Mail

In development, emails are written to `storage/logs/laravel.log`:

```env
MAIL_MAILER=log
```

For a real SMTP server (e.g. Mailtrap or Mailgun):

```env
MAIL_MAILER=smtp
MAIL_HOST=smtp.mailtrap.io
MAIL_PORT=587
MAIL_USERNAME=your-username
MAIL_PASSWORD=your-password
MAIL_FROM_ADDRESS="noreply@copyus.shop"
MAIL_FROM_NAME="${APP_NAME}"
```

## Database

### Migrations

All schema changes are managed through Laravel migrations:

```bash
php artisan migrate           # Run pending migrations
php artisan migrate:status    # List migration state
php artisan migrate:rollback  # Roll back last batch
php artisan migrate:fresh     # Drop all tables and re-run (destroys data)
php artisan migrate:fresh --seed  # Fresh migrate + seed demo data
```

### Creating the SQLite file

If you see "database file not found", create it manually:

```bash
touch database/database.sqlite   # macOS/Linux
# Windows PowerShell:
New-Item -ItemType File database/database.sqlite
```

## Queue Worker

Some features (notifications, abandoned-cart emails) depend on the queue. In development, run:

```bash
php artisan queue:work
```

To process all pending jobs and stop:

```bash
php artisan queue:work --once
```

## Creating an Admin User

After seeding or running fresh migrations, create an admin account via Tinker:

```bash
php artisan tinker
```

```php
\App\Models\User::create([
    'name'     => 'Admin',
    'email'    => 'admin@example.com',
    'password' => bcrypt('password'),
    'role'     => 'admin',
]);
```

Then log in at [http://localhost:8000/login](http://localhost:8000/login).

The admin panel is at [http://localhost:8000/admin](http://localhost:8000/admin).

## Development Workflow

### Running both servers at once

Open two terminals:

```
Terminal 1: php artisan serve
Terminal 2: npm run dev
```

Or use a tool like `concurrently` (already a dev dependency):

```bash
npx concurrently "php artisan serve" "npm run dev"
```

### Code Style

PHP code style is enforced by [Laravel Pint](https://laravel.com/docs/pint):

```bash
./vendor/bin/pint           # Fix all files
./vendor/bin/pint --test    # Check without fixing
```

### Tests

```bash
php artisan test                         # Run all tests
php artisan test --filter=OrderTest      # Run a specific test
php artisan test --coverage              # With coverage report
```

## Production Deployment Checklist

```bash
# 1. Install dependencies (no dev)
composer install --no-dev --optimize-autoloader

# 2. Environment
cp .env.example .env
php artisan key:generate
# Fill in all production values in .env

# 3. Database
php artisan migrate --force

# 4. Cache config & routes
php artisan config:cache
php artisan route:cache
php artisan view:cache

# 5. Build assets
npm ci
npm run build

# 6. Storage symlink
php artisan storage:link

# 7. Queue worker (use Supervisor or systemd)
php artisan queue:work --daemon
```

Set `APP_ENV=production` and `APP_DEBUG=false` in the production `.env`.

## Troubleshooting

| Problem | Fix |
|---|---|
| `Class not found` errors | Run `composer install` |
| `Mixed content` / assets 404 | Run `npm run build` |
| Database not found | Create `database/database.sqlite` then `php artisan migrate` |
| Notifications not sending | Run `php artisan queue:work` |
| Permission errors on `storage/` | `chmod -R 775 storage bootstrap/cache` |
| Translatable field returns `null` | Pass the locale: `->getTranslation('name', 'ca')` |
| `APP_KEY` missing | Run `php artisan key:generate` |
